AstraSync AI/ML Services Analysis

Author

John Carl Tagbor | Molly McDade | Ann Gray Perdue | Luke White

Summary

AstraSync offers an AI/ML service contract for 18 months, and is in negotiations about billing. Their first offer had two counteroffers from the client, one being $152,000/month and the other being $125,000/month plus an $800,000 bonus at the end of the contract if implementation is successful (70% chance). If AstraSync does not accept either alternative, the project will be opened to a RFP where different companies will compete for the project. AstraSync is set on bidding $150,000/month on the project, and estimates a 40% chance they’ll win. The terms of the RFP include payments of the revenue bid amount to the winning bidder at the end of each month, plus a gain-share reward at the end of month 18 depending on the amount saved for the client.

Analysis

To determine the best solution for AstraSync, Team 16 Consulting used the principles of Decision Trees. Decision trees incorporate expected values to quantify the potential outcomes of different decisions whilst factoring probabilities with each possible scenario. This helps AstraSync to weigh the financial or practical impact of the various options, allowing for a more objective comparison of risks and rewards when making informed choices.

After careful assessment of the business problem, we have established that the three alternatives available to AstraSync are as follows;

  • OPTION A: Accept equal payments of $152,000 every month for 18 months.

  • OPTION B: Accept monthly payments of $125,000 for 18 months plus a $800,000 bonus paid at the end of month 18 if the implementation is successful and outperforms significantly a benchmark provided by the client.

  • OPTION C: Participate in the bid with $150,000 for 18 months with the possibility of a gain share bonus based on savings made after successfully implementing the AI/ML solution.

Option A

Option A is to accept equal payments of $152,000 every month for 18 months. Considering that the project cost is $120,000 a month, this results in a profit per month of $32,000. Using a discount factor of 0.015 over the 18 months, the present value of Astrasync’s earnings is $501,522. Below is the computation to derive this result with variable name, “PVContrA”.

#Cost for AstraSync

ProjCost <- 1200*100*18      #2.16 million
ProjCostMonth <- 1200*100    #120k

# AstraSync's 1st proposal

FirstProposal <- 160000*18                  #2.88 mil
ContrMonthly <- 160000-ProjCostMonth        #40k/month
ContrFull <- ContrMonthly * 18              #720k
MonthlyDiscountRate <- 0.015
PVContr <- round(ContrMonthly*((1-(1+0.015)**-18)/0.015))   

# PVContr = 626,902 - how much they'd make given their first proposal

# Client Option A proposal ($152k monthly for 18 months)

MonthlyOptA <- 152000
ContrMonthlyA <- MonthlyOptA-ProjCostMonth                     #32k
PVContrA <- round(ContrMonthlyA*((1-(1+0.015)**-18)/0.015))    #501,522

Option B

Option B involves receiving monthly payments of $125,000 for 18 months, along with a potential $800,000 bonus at the end of the 18th month if the implementation is successful. If the implementation fails, AstraSync’s expected value is $102,447, with a present value of $78,363. However, if the implementation succeeds, AstraSync will earn the $800,000 bonus, in addition to the $102,447, bringing the total to $902,447. This total, when discounted to its present value, is $690,292.

Given a 70% probability of success where AstraSync earns $902,447 and a 30% probability of failure where AstraSync earns only $102,447, the expected value of Option B is $506,713. The detailed calculations are provided in the script below.

# Client Option B proposal ($125k monthly for 18 months + 800,000 bonus if successful. 70% chance of bonus)

MonthlyOptB <- 125000
ContrMonthlyB <- MonthlyOptB-ProjCostMonth                     #5k
PVContrB <- round(ContrMonthlyB*((1-(1+0.015)**-18)/0.015))    #78k
PVBonus <- round(800000/((1+0.015)**18))                       #611,929
PVContrB_withBonus <- PVContrB+PVBonus                         #690,902
PVContrB_withoutBonus <- PVContrB                              #78,363
EVContrB <- (PVContrB_withBonus*0.7) + (PVContrB_withoutBonus*0.3)   #506,713

Option C

In Option C, AstraSync participates in the bid and if unsuccessful, end up with nothing. However if successful, ends up with $614,681 and a potential to earn a bonus if the implementation of the project yields significant cost savings. The bonus earning opportunities were estimated with a triangular distribution off cost savings to be a low of $3.2M, high of $12.8M, and most likely situation of $5.6M. In the low case of $3.2M, Astrasync does not earn a bonus because the cost savings is less than $4M. The Expected value is therefore $614,681 which transalates to a present value of $470,177. If the solution generates a cost savings of $5.6M, Astrasync gets a bonus of $320,000 according to the bonus policy. All together this implies an expected value of $934,681 which translates to a present value of $714,949. If the solution generates a cost savings of $12.8M, Astrasync earns a bonus of $4.1M according to the bonus policy. All together, AstraSync earns an expected value of $4,694,681 with a present value of $3,591,016. The code for generating these results is shown below.

#Cost for AstraSync

ProjCost <- 1200*100*18      #2.16 million
ProjCostMonth <- 1200*100    #120k

# AstraSync rejects - Option C, need a histogram as well

MonthlyOptC <- 150000                        #To be paid to winning bidder monthly
ContrMonthlyC <- MonthlyOptC-ProjCostMonth   #30k

# Base value of 150k bid - 120k cost discounted for 18 months. Also the low savings since there is no bonus

SavingsLow <- 3200000
PVContrC <- round(ContrMonthlyC*((1-(1+0.015)**-18)/0.015))   #470,177

# Most Likely Savings

SavingsML <- 5600000
BonusML <- (SavingsML - 4000000)*0.2
PVBonusML <- round((BonusML/((1+0.015)**18)))            #244,772
PVTotalML <- PVContrC + PVBonusML                        #714,949

# High Savings

SavingsHigh <- 12800000
BonusHigh <- (SavingsHigh - 8000000)*0.6 + 1200000
PVBonusHigh <- round(BonusHigh/(1.015**18))           #3.12mil
PVTotalHigh <- PVContrC + PVBonusHigh                 #3,591,016

############### Triangular Plot
library(triangle)
library(tidyverse)
data <- rtriangle(10000000,SavingsLow,SavingsHigh,SavingsML)

gainSharesSims <- data.frame(value = case_when(data < 4000000 ~ 0,
                            data < 6000000 ~ 0.2*(data-4000000),
                            data < 8000000 ~ 400000 + 0.40 * (data - 6000000),
                            TRUE ~ 1200000 + 0.60 * (data - 8000000)))

expGainShare <- mean(gainSharesSims$value)
PVGainShare <- expGainShare / (1 + 0.015)**18
PVContrC_Succ <- PVContrC + PVGainShare
ExpValueC <- 0.4 * (PVContrC + PVGainShare)  ###Expected value for C

# Plot the Simulation
library(ggplot2)
ggplot(gainSharesSims, aes(x = value)) + 
  geom_histogram(bins = 8, color = "white") + 
  labs(title = "Triangular Distribution of Share of Savings", subtitle = "1000000 Simulations") +
  geom_vline(aes(xintercept = mean(value)), linetype = "solid", color = "orange", lwd =1) + 
  geom_vline(aes(xintercept = median(value)), linetype = "solid", color = "purple", lwd=1)

Conclusion

Based on the expected values for the various options, we recommend that AstraSync goes with Option C with an expected payoff of $509,718.

In this case, there is a 60% chance that they don’t win the Bid and that would mean, they end up with nothing.

Our recommendation therefore is that AstraSync takes option B and does everything possible to ensure that the solution is effective so that they can get the bonus and a total expected value of $506,713.
